Prompt Terminal
Prompt Classification
提示词的两大分类
User Prompt
用户提示词
用户对模型发出的具体指令或问题

示例："帮我推荐附近的餐厅"

• 随对话动态变化
• 表达具体需求与意图
• 每次交互可以不同
System Prompt
系统提示词
提供环境背景与角色定义

示例："你是专业的营养师"

• 定义 AI 的"人设"
• 减少重复信息输入
• 提高对话一致性
Best Practices
Prompt 设计四大原则
1
简洁明了 — 避免冗余语境，直击核心需求
2
正向提问 — 告诉模型"要做什么"而非"不要做什么"
3
逐步引导 — 引导分步思考，让推理过程更清晰
4
ICL 上下文学习 — 提供少量示例，帮助模型泛化输出
推荐使用 Markdown 语法 格式化 Prompt

Quadrant Deep Dive
四象限详解：协作策略
🛎️ 助理模式
场景：日常执行类任务

核心策略：
• 指令简洁直接
• 重点是精准派活
• 明确边界和约束

示例："帮我生成一个贪吃蛇小游戏"
🔬 共创模式
场景：AI缺乏你的领域知识

核心策略：
• 先发散后收拢（双钻模型）
• 人负责调研与收集
• AI 负责提炼与总结

技巧：提供大量样本 → 上下文内学习
Double Diamond

Key Takeaways
核心知识总结
模型基础
大模型 = 泛化 + 涌现
核心能力是泛化与涌现。涌现包含 ICL、指令跟随、CoT。但存在四大局限。
提示词
Prompt = 编码意图的语言
分为用户提示词和系统提示词。设计遵循简洁、正向、逐步引导、ICL 四大原则。
协作框架
四象限 = 助理/老师/共创/探索
基于乔哈里视窗，根据双方知识映射四种协作模式。
上下文
Context = 聊天记录的系统工程
通过人设定义、样本投喂、结构化输入、渐进对话提升效果。
