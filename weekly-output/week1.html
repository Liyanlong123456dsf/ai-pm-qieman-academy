课程 002 · Day 1
场景思维 — 产品经理的第一课
场景描述公式
用户（谁）+ 场景（在哪）+ 痛点（怎么痛）
场景 = 讲故事：主人公 + 情境 + 冲突 + 解决方案
"从现在开始，张口就要提场景，用语言表达出来"
1
BRD文档（Business Requirement Document）：分清做什么与不做什么，明确技术路线
2
产品分析画布：结构化拆解产品的分析工具，适用竞品研究和自身产品规划
3
技术评审：评估技术可行性是产品方案落地的关键环节，PM需具备基础技术语感

课程 004 · Day 2
POC验证与MVP方法论
🧪
POC — 概念验证
回答："做不做得出来？"
流程：想清楚验证什么 → 定义成功标准 → 快速试验 → 结果决策
可跳过：技术方案已成熟时
举例：用Dify验证RAG能否满足客服场景准确率
🚀
MVP — 最小可行产品
回答："做出来有没有人用？"
目标：用最低成本验证市场需求
可跳过：市场需求已验证时
举例：上线简化版智能客服，观察使用率和满意度
"POC回答'做不做得出来'，MVP回答'做出来有没有人用'—— 两者缺一不可"
课程 005 · Day 2
LLM推理流程 — 一句话如何被理解
❓
Query
用户提问
→
✂️
Tokenization
分词拆解
→
🔤
Token
最小语义单元
→
📐
Embedding
向量化表示
→
💡
Semantics
语义理解
通俗理解：你说了一句话 → 拆成小碎片（Token）→ 变成数学向量（Embedding）→ 计算距离找语义关系 → 生成最可能的下一个词

关键认知：LLM本质是概率预测——预测下一个最可能出现的词，而非"理解"

Transformer对比：CNN逐步扫描局部（像看拼图碎片），RNN按顺序但记忆有限，Transformer全局并行理解（像总指挥俯瞰全局）

课程 007 · Day 3
大模型训练 — 两阶段流程
第一阶段
预训练 Pre-training
方式：无监督学习 — 从数据中自己找规律
数据：维基百科、论文、图书、网页
三要素：数据采集 + 注意力机制 + Transformer
类比：小孩通过大量阅读获得基础语感
第二阶段
微调 SFT
全称：Supervised Fine-Tuning
方式：有监督 — 人工标注什么是好回答
目标：针对对话、代码、医疗等垂直领域优化
类比：从"什么都懂一点"变成"某领域精通"
预训练（通用能力）→ 微调（专项能力）→ 部署上线
课程 007 · 深化
泛化能力与涌现能力
📉
欠拟合
学习不足，做不到举一反三
类比：只看了目录就考试
📈
过拟合
过度学习，只会背答案
类比：背了全部习题不懂原理
✨
泛化
学到抽象知识，迁移到新任务
类比：真正理解，什么题都能做
涌现能力 Emergence
模型规模达到阈值后突然涌现新能力
三大子能力：小样本学习 Few-shot ICL + 指令跟随 + 思维链推理 COT
课程 007 · 补充
大模型四大局限性 — PM必须知道
⏰
时效性局限
训练数据有截止日期，不知道最新事件
解决：RAG检索增强 + 联网搜索
👻
模型幻觉
基于概率预测，可能"一本正经地胡说八道"
解决：人工审核 + 事实核查 + RAG
🔬
专业性不足
公开数据训练，垂直领域知识薄弱
解决：专业数据微调 + 领域知识库
🔒
安全性风险
可能生成有害/偏见/隐私泄露内容
解决：RLHF对齐 + 过滤 + 私有化部署
课程 007-2 · Day 3
Prompt工程 — 人机沟通的桥梁
核心公式
效果 = 模型能力 × 任务设计 × Prompt质量
1
Prompt本质：人与AI对话的桥梁，承担"编码"和"解码"角色
2
User Prompt：用户直接输入的指令 — "你说的话"
3
System Prompt：设定对话背景和AI行为 — "你给AI的角色和规则"
4
同一句话不同System Prompt：
"我饿了" → 宿舍：室友推荐外卖 / 春节：妈妈做饭 / 情侣：一起出去吃
Prompt 设计方法论
六大设计原则
✨
简洁明了
避免冗余，一个Prompt只解决一个问题
👍
正向描述
告诉AI"做什么"而非"不做什么"
🪜
逐步引导
Chain-of-Thought，复杂问题分解
📝
提供样本
ICL上下文学习：给1-3个示例
📑
Markdown格式
结构化格式显著提升模型理解
🎭
角色设定
通用结构 / 任务执行 / 角色扮演
课程 007-2 · 核心模型
乔哈里视窗 — 人机协作四象限
🔓
公开象限：我知·AI知
AI作助理
精准指令让AI执行
场景：让AI写你熟悉领域的报告
🔒
隐私象限：我知·AI不知
投喂上下文
提供独有知识让AI理解
场景：提供内部文档后再提问
🔍
盲点象限：AI知·我不知
AI作老师
向AI提问学习新知识
场景：请AI解释注意力机制
🌌
潜能象限：双方都不知
共创探索
先发散再收拢探索未知
场景：与AI头脑风暴新产品方向
课程 007-2 · 进阶
双钻模型与上下文工程
💎
双钻模型
第一颗钻石：发散 — AI负责生成大量可能性
第二颗钻石：收拢 — 人负责筛选和决策
核心：AI不替代判断力，而是放大视野
🧩
上下文工程
定义：系统化管理输入给模型的上下文
ICL：将示例放入上下文让模型学习
Markdown格式化 → 模型理解显著提升
这是AI PM的核心技能之一
"大模型效果 = 模型能力 × 任务设计 × Prompt质量"
