<div class="eyebrow">PHASE 0 · 笔记 → 数据</div>

# 第一步：把课堂内容变成结构化数据

<div class="phase-badge phase-0">PHASE 0 · 基础准备</div>

<div class="two-col">
<div>

### 我做了什么

1. **听课时实时记录**关键概念、老师原话、作业要求
2. 课后把散乱笔记**整理成固定格式**：
   - 课程 ID / 标题 / 标签
   - 核心知识点（points 数组）
   - 关键要点（keyPoint）
   - 作业内容（homework）
   - 关联 PM 节点（relatedNodes）
3. 用 **JS 对象字面量**存储，不用数据库

### AI 协作方式

<div class="chat-label">我说：</div>
<div class="chat-human">帮我把这节课的笔记整理成结构化的 JS 数据，格式参考模板：{id, title, tags, points[], keyPoint, homework}</div>

<div class="chat-label">AI 做的：</div>
<div class="chat-ai">把散乱笔记提炼为 10 个知识点，匹配 PM 流程节点，生成完整数据结构</div>

</div>
<div>

### 数据结构示例

```javascript
{
  id: '001',
  title: 'IDE工具安装与AI编程初体验',
  tags: ['工具', '实战'],
  relatedNodes: [
    { id: 6, title: '开发跟进与迭代',
      why: '开发工具与效率提升' }
  ],
  points: [
    'IDE工具选型：Cursor 与 Trae',
    'AI辅助编程：自然语言生成代码',
    'Bug反馈思维：明确描述问题现象',
  ],
  keyPoint: 'Bug反馈思维',
  homework: '用AI生成创意小游戏'
}
```

<div class="tip-box" style="margin-top: 10px;">
  <strong>心法：</strong>数据结构先于 UI。先想清楚"数据长什么样"，再考虑"界面怎么展示"。
</div>

</div>
</div>

<div class="two-col">

### 我跟 AI 说了什么

<div style="display: flex; flex-direction: column; gap: 6px; margin-top: 8px;">
<div class="chat-human">我在学 AI 产品经理课，想做一个交互式知识库网页。要求：纯前端单文件、力导向图谱、课程笔记系统、PM 全流程节点、暗色 Apple 风格</div>
<div class="chat-ai">建议用 TailwindCSS CDN 做样式，Canvas 2D 做力导向图渲染，所有数据内嵌 JS，无需构建工具...</div>
</div>

### 第一版核心功能

- **力导向知识图谱** — Canvas 2D 物理模拟
- **11 个 PM 核心节点** — 需求→设计→开发→上线→复盘
- **课程笔记系统** — 按周/天/课组织
- **暗色/亮色主题** — 一键切换
- **进度追踪** — localStorage 持久化

</div>
<div>

### 技术选型决策

| 选项 | 决策 | 原因 |
|------|------|------|
| 框架 | Vanilla JS | 零依赖，双击可用 |
| 样式 | TailwindCSS CDN | 快速原型，无构建 |
| 图谱 | Canvas 2D | 性能好，自定义高 |
| 数据 | JS 对象内嵌 | 单文件，无需后端 |
| 存储 | localStorage | 笔记/进度持久化 |

<div class="warn-box" style="margin-top: 12px;">
  <strong>为什么选单文件 HTML？</strong><br/>
  1. 分享方便 — 发一个文件就能用<br/>
  2. 无需构建 — 没有 npm/webpack<br/>
  3. 部署简单 — 丢到任何静态托管<br/>
  4. 学习成本低 — 一个文件看全貌
</div>

</div>
</div>
